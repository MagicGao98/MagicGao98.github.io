Linux Tips

Linux 知识

  1. 把用户加入root组

    1. 切换到root用户，运行visudo命令
    2. 在配置文件中找到root ALL=(ALL) ALL，在下面一行添加 xxx ALL=(ALL) ALL 其中xxx是你要加入的用户名称
    3. 保存退出就OK了

  2. shadowsocks一键配置包下载与安装

      git clone -b master https://github.com/flyzy2005/ss-fly
      ss-fly/ss-fly.sh -i major123 1024

  3. shadowsocks一键包的相关操作

      修改配置文件：vim /etc/shadowsocks.json
  停止ss服务：ssserver -c /etc/shadowsocks.json -d stop
  启动ss服务：ssserver -c /etc/shadowsocks.json -d start
  重启ss服务：ssserver -c /etc/shadowsocks.json -d restart
  卸载ssr服务:
  ./shadowsocksR.sh uninstall

  4. vim 修改只读文件修改完权限不够

      :w !sudo tee %

  5. 一键开启BBR加速

      ss-fly/ss-fly.sh -bbr

    判断是否成功：

      sysctl net.ipv4.tcp_available_congestion_control

    返回如下则为成功yum

      net.ipv4.tcp_available_congestion_control = bbr cubic reno

  6. Linux 换源

    1. Centos 换源
      1. 进入yum源配置文件cd /etc/yum.repos.d
      2. 备份一份当前源，以防出错 mv ./Centos-Base.repo ./Centos-Base-repo.bak
      3. 下载网易163的源 wget http://mirrors.163.com/.help/CentOS7-Base-163.repo
      4. 清理一下旧包 yum clean all
      5. 把下载下来文件 CentOS7-Base-163.repo设置成为默认源 mv CentOS7-Base-163.repo CentOS-Base.repo
      6. 生成缓存即可用163源了 yum makecache
      7. 现在我们就可以用163源来安装软件了！ 如下我们安装一个php
        yum install php
      8. 查看一下，可以看到已经安装好了：yum list | grep php
    2. Ubuntu 换源
      1. 备份原来的源：sudo cp /etc/apt/sources.list /etc/apt/sources_init.list

  7. Linux软件卸载
    rpm -qa | grep nginx #比如这里就是卸载nginx，先查看是否安装有nginx
    yum erase nginx -y #CentOS用yum来卸载软件

  8. CentOS7 配置中文输入和显示中文
    在shell中输入localectl命令，观察显示LANG这个参数是什么
    如果不是zh_CN.utf-8就需要配置
    使用命令：localectl set-locale LANG=zh_CN.utf8

  9. Linux网络配置
    首先检查当前网络是否配置好，可用：ping 8.8.8.8
    再检查网络设备是否存在：ifconfig
    ifconfig devicename ip/24：设定IP
    ifconfig devicename down：关闭
    ifconfig devicename up：打开
    ip addr：检测和设定网络接口
    ip addr show：检测
    ip addr add ip/24 dev devicename：设定IP
    管理网络配置文件：
    vim /etc/sysconfig/network-scripts/ifcfg-xxx

    DEVICE=eth0
ONBOOT=yes
IPADDR=172.25.254.100
NETMASK=255.255.255.0
BOOTPROTO=none
NAME=westos

修改完环境变量后可以通过souce命令使之生效

  2. 修改主机名： sudo vim /etc/hostname
    hosts文件作用：建立域名与IP的映射
    修改hosts： sudo vim /etc/hosts

  1. Linux系统下文件夹及其作用
    /bin 放置执行文件目录，root和一般用户都可执行的名令
    /boot 放置开机使用到的文件，包括内核文件和开机菜单以及开机所需要的配置文件
    /dev 存放任何的设备和接口设备，linux下都是以文件的形式存在于这个目录下，通

            过访问文件访问这些设备

    /etc 放置系统几乎所有的系统配置文件，包括账号密码文件、各个服务的起始文件等

            建议不要放置任何可执行文件

    /home 系统默认的用户诸位文件夹，创建一个用户

            就会在这个目录下创建一个用户家目录

    /lib 放置系统函数库
    /media media即是媒体，存放可以删除的设备，包括软盘、光盘、DVD等临时挂放在这里
    /mnt 挂载额外的设备，早期与/media作用相同
    /opt

  2. Linux添加环境变量PATH失败

    • 问题复现： 在当前用户的PATH配置文件.bash_profile对PATH进行修改，并执行source .bash_profile, 但是PATH无法生效
    • 问题原因： 每个用户都有自己的PATH配置文件,但是整个系统也有一个PATH配置文件，当在用户的配置文件中修改PATH之后，需要分别执行source .bash_profile和source /etc/profile之后PATH才生效

  3. Linux命令别名

    别名暂时设置，如：alias g++11="g++ -std=c++11"

    别名永久设置：

    修改：/root/.bashrc
添加：alias g++11="g++ -std=c++11" 这样的命令
source使配置文件重新生效：$source /etc/profile
