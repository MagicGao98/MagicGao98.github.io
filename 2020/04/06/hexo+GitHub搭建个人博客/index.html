    hexo+GitHub搭建个人博客

    前期准备

    • 安装 node.js

    • 安装 git

    • 创建博客仓库：用户名.github.io

    • 域名注册绑定（可选）：godaddy 或者 阿里云

    • 配置 ssh key

    使用 hexo 搭建博客

    安装 hexo

    npm install -g hexo

    初始化

    # 在合适的位置建立 hexo 目录文件
mkdir hexo
cd hexo
# 初始化 hexo
hexo init
# hexo 目录下出现hexo博客基础文件
hexo g # 将markdown文件生成静态网页文件
hexo s # 启动hexo本地预览服务，在本地访问搭建的blog url: http://localhost:4000

    注意：如果本地预览失败，可考虑端口冲突影响

    修改主题

    hexo官方主题：https://hexo.io/themes/

    cd hexo
# clone 自己喜欢的主题，并放到themes目录下
git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia
# 修改全局配置文件_config.yml(即hexo直接目录下的_config.yml)
vim _config.yml
# 将 theme: xxx 改为 theme: yilia (yilia是自己下载的主题名，在hexo/theme/中选择)
hexo clean # 清理hexo/public/下的内容（存放之前生成的静态文件）
hexo g # 重新生成静态文件

    写博客&管理博客

    • 原始方法

      博客的markdown源文件存放在 hexo/source/_posts/ 目录下

      hexo new "title" 
# 会在 hexo/source/_posts/ 目录下生成 "title.md" 文件
# 也可以直接将markdown源文件复制到 hexo/source/_posts/ 目录下
hexo g

      markdown博文源文件一般完整格式

      ---
title: postName #文章页面上的显示名称，一般是中文
date: 2013-12-02 15:30:16 #文章生成时间，一般不改，当然也可以任意修改
categories: 默认分类 #分类
tags: [tag1,tag2,tag3] #文章标签，可空，多标签请用格式，注意:后面有个空格
description: 附加一段文章摘要，字数最好在140字以内，会出现在meta的description里面
---

以下是正文

    • Hexo Admin插件管理（推荐）

      安装插件

      npm install --save hexo-admin

      启动本地预览

      hexo server -d

      即可在 localhost:4000/admin/ 中编辑博文

      注意：博文中插入图片可以统一将图片放置在hexo/source/images/目录下，然后使用![](/images/image.jpg) 来引用

    部署博客

    前期准备中创建好的 用户名.github.io 仓库就是个人博客的部署目的地，此仓库可以被解析和直接访问，是静态个人博客搭建好地方

    安装所需插件：

    nmp install hexo-deployer-git --save

    修改全局配置文件_config.yml 中有关deploy的部分，示例如下：

    deploy:
  type: git
  repository: git@github.com:用户名/用户名.github.io.git
  branch: master

    部署到GitHub仓库

    hexo d

    注意：(1) hexo部署时会把以前的所有代码都删掉，虽然有版本管理，但最好还是学会本地备份。(2) hexo 部署时本地文件hexo/source/目录下的所有文件都会被原样复制到仓库里的public目录下

    常用 hexo 命令

    # 常见命令
hexo new "pastName"  # 创建博文
hexo new page "pageName"  # 创建页面 区别于 hexo new 会以pagename创建目录文件，里面包含 index.md 源文件
hexo generate  # 生成静态页面至 hexo/public 目录
hexo server  # 开启本地预览
hexo deploy  # 部署到GitHub
hexo help  # 查看帮助
hexo version  # 查看hexo的版本
# 缩写
hexo n == hexo new
hexo g == hexo generate
hexo s == hexo server
hexo d == hexo deploy
# 组合命令
hexo s -g  # 生成并本地预览
hexo d -g  # 生成并上传
